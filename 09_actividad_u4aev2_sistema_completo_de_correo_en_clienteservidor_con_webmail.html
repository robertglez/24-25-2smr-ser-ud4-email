09| Actividad U4.AEV2. Sistema completo de correo en cliente-servidor con webmail

Actividad Evaluable: Configuración de Servidor de Correo con Roundcube

📚 Detalles

Módulo: Servicios en Red

Unidad Didáctica: Servicios de Correo Electrónico

Nivel: 2º SMR

Duración: 3 sesiones (3 horas aprox.)

🎯 Objetivo

Los alumnos instalarán y configurarán un servidor de correo con Postfix (SMTP), Dovecot (IMAP/POP3) y Roundcube (webmail), y conectarán clientes de correo (Thunderbird y navegador web) para enviar y recibir mensajes.

🛠 Materiales Necesarios

  • Máquinas virtuales con Ubuntu Server 22.04 LTS (servidor) y Ubuntu Desktop (cliente)
  • Conexión a internet para descargar paquetes
  • Acceso root o usuario con sudo

🔧 Pasos de la Actividad

1. Preparación e instalación de la máquina virtual con Ubuntu Server 22.04

1.1. Crear la máquina virtual en VirtualBox

  1. En VirtualBox, haz clic en Nueva.

  2. Asigna un nombre a tu VM, por ejemplo, ServidorCorreo.

  3. Selecciona Linux como tipo de sistema operativo y Ubuntu (64-bit) como versión.

  4. Asigna una cantidad de memoria RAM adecuada, p. ej. 2 GB (2048 MB) o más.

  5. Crea un disco duro virtual de al menos 20 GB.

  6. En Configuración de la VM, en el apartado Red, configura el Adaptador de red en modo Red interna o Adaptador puente, dependiendo de tus necesidades. Es importante que las máquinas (servidor y cliente) estén en la misma red.

1.2. Instalar Ubuntu Server 22.04

  1. Descarga la ISO de Ubuntu Server 22.04 desde la página oficial de Ubuntu.

  2. Inicia la VM de ServidorCorreo y selecciona la ISO descargada para arrancar desde ella.

  3. Sigue las instrucciones del instalador:

    • Selecciona Instalar Ubuntu Server.

    • Elige el idioma y la distribución de teclado.

    • Configura la dirección IP (estática o DHCP según convenga). Para un servidor de correo, suele ser recomendable una IP estática.

    • Crea tu usuario administrador y contraseña.

    • Asigna un nombre de equipo (hostname), por ejemplo mailserver.

    • Completa la instalación y reinicia.

1.3. Actualizar el sistema

Tras iniciar sesión, asegúrate de que tu servidor Ubuntu está actualizado:

sudo apt update
sudo apt upgrade -y

2. Configuración de Postfix

2.1. Instalar Postfix

sudo apt install postfix -y

Durante la instalación, te pedirá que elijas el tipo de configuración de Postfix. Selecciona "Sitio de Internet" (Internet Site) y, cuando se solicite, introduce el nombre de correo (system mail name) que coincida con tu dominio o subdominio deseado (p. ej., midominio.local o mail.local).

Nota: En un entorno real, conviene apuntar el DNS al servidor y disponer de un dominio con registros MX. Para una práctica local, bastará con usar un dominio ficticio (p. ej. ejemplo.local).

2.2. Verificar el archivo principal de configuración

Abre /etc/postfix/main.cf para comprobar que coincide el nombre de host y dominio, y que la línea myhostname y mydomain estén bien configuradas. Un ejemplo:

myhostname = mail.ejemplo.local
mydomain = ejemplo.local
myorigin = $mydomain
mydestination = $myhostname, localhost.$mydomain, localhost, $mydomain
relayhost =
mynetworks = 127.0.0.0/8, [::1]/128
home_mailbox = Maildir/
smtpd_banner = $myhostname ESMTP

Si realizas cambios, reinicia Postfix:

sudo systemctl restart postfix

3. Configuración de Dovecot

3.1. Instalar Dovecot

sudo apt install dovecot-core dovecot-imapd dovecot-pop3d -y

3.2. Configurar Dovecot para Maildir

  1. Edita el archivo /etc/dovecot/dovecot.conf y asegúrate de que el protocolo deseado (IMAP y/o POP3) está habilitado.

  2. En /etc/dovecot/conf.d/10-mail.conf, revisa la ruta de almacenamiento. Se recomienda Maildir:

mail_location = maildir:~/Maildir

  1. En /etc/dovecot/conf.d/10-auth.conf, habilita la autenticación típica:

disable_plaintext_auth = no
auth_mechanisms = plain login

  1. Revisa /etc/dovecot/conf.d/10-master.conf para que el servicio de autenticación (auth) escuche adecuadamente.

Tras editar, reinicia Dovecot:

sudo systemctl restart dovecot

4. Creación de cuentas de correo

Para este ejemplo, crearemos dos usuarios locales, usuario1 y usuario2. El correo se almacenará en ~/Maildir (Maildir) de cada usuario.

sudo adduser usuario1
# Se solicitará una contraseña y datos opcionales.

sudo adduser usuario2
# Se solicitará una contraseña y datos opcionales.

Cada usuario tendrá su dirección de correo: usuario1@ejemplo.local y usuario2@ejemplo.local, asumiendo que tu dominio es ejemplo.local.

Nota: Si no deseas cuentas de sistema, se puede configurar Postfix y Dovecot con usuarios virtuales, pero para una práctica básica, los usuarios del sistema son suficientes.

5. Instalación y configuración de Roundcube

5.1. Instalar servidor web y PHP

sudo apt install apache2 php php-cli php-curl php-mbstring php-xml php-zip php-intl php-sqlite3 -y

(Se puede usar Nginx en lugar de Apache si se prefiere.)

5.2. Instalar Roundcube

sudo apt install roundcube roundcube-core roundcube-mysql roundcube-plugins -y

El instalador preguntará por la configuración de la base de datos. Si deseas usar SQLite para simplicidad, elige esa opción. Si usas MySQL/MariaDB, asegúrate de crear la base de datos y el usuario adecuados.

5.3. Verificar la configuración de Roundcube

  1. Comprueba el archivo de configuración de Roundcube, normalmente en /etc/roundcube/config.inc.php.

  2. Ajusta el parámetro default_host al nombre de tu servidor IMAP (p. ej. mail.ejemplo.local).

  3. Ajusta smtp_server a tu servidor SMTP (p. ej. mail.ejemplo.local).

5.4. Comprobar acceso web

  1. Abre un navegador en la misma red que el servidor.

  2. Accede a http://<IP_o_HOST_DEL_SERVIDOR>/roundcube.

  3. Inicia sesión con el usuario y contraseña (p. ej. usuario1 / su contraseña).

Si todo está bien, Roundcube mostrará la bandeja de entrada.

6. Prueba desde máquina virtual cliente

6.1. Crear la VM cliente en VirtualBox

  1. En VirtualBox, haz clic en Nueva y nómbrala ClienteCorreo.

  2. Selecciona tipo Linux, según prefieras para probar. Puede ser un Ubuntu Desktop, u otra distro.

  3. Configura el adaptador de red de forma que coincida con el modo de red del servidor (Red interna o puente) para que estén en la misma subred.

  4. Instala el SO deseado.

6.2. Acceder a Roundcube desde el cliente

  1. En el cliente, abre un navegador.

  2. Escribe la IP del servidor (p. ej. http://192.168.1.100/roundcube) o el hostname (http://mail.ejemplo.local/roundcube).

  3. Inicia sesión con usuario1 y comprueba el envío y recepción de correos (por ejemplo, envía un correo a usuario2).

6.3. Instalar Thunderbird y configurar las cuentas

En tu cliente Linux, instala Thunderbird:

sudo apt update
sudo apt install thunderbird -y

6.3.1. Configurar usuario1 en Thunderbird

  1. Abre Thunderbird.

  2. Cuando pida crear una cuenta, selecciona la opción de Usar una cuenta existente.

  3. Introduce:

    • Nombre: Usuario 1

    • Dirección de correo: usuario1@ejemplo.local

    • Contraseña: la que se configuró para el usuario1.

  4. Thunderbird intentará detectar automáticamente los parámetros IMAP/SMTP.

  5. Ajusta manualmente si es necesario:

    • Servidor de entrada (IMAP): mail.ejemplo.local (puerto 143 o 993 si usas SSL).

    • Servidor de salida (SMTP): mail.ejemplo.local (puerto 25 o 587 si has configurado STARTTLS).

  6. Guarda la configuración.

  7. Envía un correo a usuario2@ejemplo.local y revisa la bandeja de envío y recepción.

6.3.2. Configurar usuario2 (opcional)

Repite los pasos anteriores para usuario2 y comprueba que también se puedan enviar y recibir correos.

7. Comprobaciones finales

  1. Envía correos de usuario1 a usuario2 y viceversa desde Roundcube y Thunderbird.
  2. Verifica que llegan a la bandeja de entrada correspondiente y se pueden leer.
  3. Prueba la interfaz de Roundcube para leer y contestar correos.
  4. Observa los registros de Postfix y Dovecot en /var/log/mail.log y /var/log/mail.err si algo no funciona.

📝 Evaluación

  • Funcionalidad (50%): Correos enviados/recibidos en Thunderbird y Roundcube
  • Configuración (30%): Archivos de configuración sin errores
  • Documentación (20%): Informe con capturas de pantalla

💡 Errores comunes y Documentación

  • Solución de errores comunes:
    • Certificados SSL: Aceptar excepciones en navegador
    • Firewall: Verificar puertos abiertos con sudo ufw status
  • Recursos: Documentación Oficial de Roundcube
